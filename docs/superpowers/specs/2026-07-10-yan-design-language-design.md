## 0. 对 `.design_library` 设计规划的审阅结论

### 0.1 保留（已经做对的事）

| 资产 | 评价 |
|------|------|
| 冷靛紫 `#4F46E5` × 暖石中性色 `#FAFAF9` | 「冷主色 + 暖底色」的张力是整套系统最有价值的决策——它天然就是**墨与纸**的关系，是东方美学可以直接生长的地基。全部色值保留。 |
| 半径三层制（控件 8 / 容器 12 / 浮层 16） | 「控件—容器—浮层」一眼可辨，符合格式塔的相似性分组。保留。 |
| 五层 whisper-quiet 暖黑阴影 | 阴影基于 `rgba(28,25,23,…)` 暖黑而非纯黑，克制且贴合对象。保留，且升格为美学原则（见 §1.4）。 |
| 36px 控件高度、4px 网格、发丝级描边 | 密度与节奏成立。保留。 |
| 文案语域（两字状态词、动宾短语、无 emoji） | 已是「文人工具」的语感。保留并纳入哲学层。 |

### 0.2 缺口（本规范要补的事）

1. **动效系统整体缺失**——UI-SPEC v1 仅 2 个 keyframe 与一句「150ms ease」，无时长/缓动 token、无编排规则、无 `prefers-reduced-motion` 策略。→ §3
2. **图谱画布无设计语言**——图谱是产品的心脏，但设计库只覆盖 dashboard 六组件；焦点模式、影响分析、状态流转这些核心场景没有视觉与动效定义。→ §4
3. **中文字体栈缺口**——设计库 caveat #6 自认：Inter 无中文字形，CN-first 产品的中文观感全交给系统缺省。→ §2.2
4. **交互状态模型分散**——hover/active/focus/disabled 各组件各写各的，没有统一的状态层。→ §5
5. **空状态 / 加载 / 错误三态未定义**。→ §6
6. **设计决策缺少心理学依据层**——token 有了，但「为什么」缺席，后续决策没有可依循的判断框架。→ §1、附录 A
7. **暗色模式组件级未验证**（设计库 caveat #5 自认）。→ §7

## 1. 设计哲学：文房四宝

LogiMap 把散落在代码、文档、会议、大脑里的业务知识，研磨成可检索、可评审、可追溯的逻辑资产——**这正是砚台的工作：把墨块研成可以书写的墨**。

设计语言命名为**「砚」**。四个意象对应四类设计对象：

| 意象 | 对应 | 设计含义 |
|------|------|----------|
| **纸** | 画布与留白 | 暖白 `#FAFAF9` 即宣纸。留白是结构，不是空缺。 |
| **墨** | 逻辑与主色 | 靛墨 `#4F46E5` 是唯一的墨色，只落在「可交互」与「被强调」处。墨分五色（焦浓重淡清）→ 主色十档的消费纪律。 |
| **笔** | 交互与动效 | 每次操作如运笔：起笔即时（<100ms 反馈）、行笔流畅（150–250ms 过渡）、收笔含蓄（ease-out 收束，无回弹）。 |
| **砚** | 工具本身 | 厚重、安静、可靠，成就书写者——UI 是逻辑内容的容器，不争抢注意力（继承 UI-SPEC v1 原则 1）。 |

### 四条美学原则（与心理学依据）

**1.1 留白即结构（间 / Ma）**
用空间分组，而非线条和框。密疏有节奏——「疏可走马，密不透风」：图谱画布、空状态、页头允许大片留白；表格、节点卡内部保持工程密度。
*依据：格式塔邻近性原则——间距本身就是分组信息；认知负荷理论——去掉的每一条分割线都是省下的一次视觉解析。*

**1.2 气韵生动（动效是气息，不是特效）**
所有运动模拟自然物的减速——叶落、水定、墨沉。ease-out 主导，无弹簧、无回弹、无旋转。动效必须携带信息（状态变化、空间来源、因果传播），纯装饰动效不做。
*依据：Doherty 阈值（<400ms 维持心流）；因果知觉——恰当的时序让用户「看见」因果而不必推理。*

**1.3 虚实相生（焦点即雾景）**
强调一个元素的方式是让其余退成虚景，而不是给它加更重的框。焦点模式中无关节点降为薄雾；模态之下页面沉降。
*依据：图形—背景组织（figure-ground）；选择性注意——降低竞争刺激比增强目标刺激更有效。*

**1.4 克制生敬（一屏一意）**
一屏至多一个强调色落点；语义色只在状态处出现；阴影静息态不高于 shadow-1。色彩的克制换来它出现时的分量。
*依据：Von Restorff 隔离效应——强调项的记忆度与稀缺度成正比；审美—可用性效应——克制的秩序感直接提升「好用」的感知。*

## 2. Token 扩展（对 `colors_and_type.css` 的增量）

### 2.1 色彩：命名层加一层「文房语义」

色值**零改动**，仅为语义色补充消费指引的助记名（用于文档与设计沟通，不新增 CSS 变量）：

| 现有 token | 助记 | 消费场景 |
|-----------|------|----------|
| primary（靛紫） | **靛墨** | 可交互、强调、选中、激活连线 |
| success（翡翠） | **石绿** | 已确认（APPROVED） |
| warning（琥珀） | **藤黄** | 评审中（REVIEW） |
| error（玫瑰） | **胭脂** | 已废弃（DEPRECATED）、破坏性操作 |
| info（天蓝） | **花青** | 提示信息 |
| neutral（暖石） | **纸灰** | 草稿（DRAFT）、全部结构色 |

### 2.2 中文排印（新增，修复设计库 caveat #6）

```css
:root {
  /* 中西文混排主栈：Inter 承担拉丁，中文按平台落到高质量黑体 */
  --font-body: 'Inter', 'PingFang SC', 'Hiragino Sans GB',
               'Noto Sans SC', 'Microsoft YaHei', sans-serif;
  --font-heading: var(--font-body);
  --font-display: var(--font-body);
  /* 衬线强调栈：全产品唯一允许出现衬线的地方是「空状态引导语」（见 §6.1） */
  --font-serif-accent: 'Noto Serif SC', 'Songti SC', 'SimSun', serif;
  --font-mono: 'JetBrains Mono', 'PingFang SC', ui-monospace, monospace;
}
```

排印纪律：

- 中文正文行高上调：body `1.55 → 1.7`，caption `1.4 → 1.6`（汉字方块字形需要更大行距呼吸；纯拉丁场景如 codeRef 保持原值）。
- 汉字**不加 letter-spacing**（破坏字形节奏）；全大写拉丁标签保留 `0.02–0.05em` 追踪。
- 中西文之间依赖字体渲染自然留隙，不手工加空格。
- 数字对齐场景（表格、版本号、计数）一律 `font-variant-numeric: tabular-nums`。

### 2.3 动效 token（新增，本规范核心之一）

时长五档，命名取书写节奏：

```css
:root {
  --motion-instant: 80ms;    /* 瞬 —— 按下反馈、取消、退出的第一帧 */
  --motion-fast: 140ms;      /* 疾 —— hover、退出动画、行内切换 */
  --motion-base: 220ms;      /* 常 —— 进入动画、展开折叠、Dialog */
  --motion-slow: 320ms;      /* 缓 —— 侧栏面板、场景级切换、云雾聚焦 */
  --motion-deliberate: 560ms;/* 凝 —— 图谱涟漪单跳、落印确认（含义重大的时刻） */

  --ease-settle: cubic-bezier(0.22, 1, 0.36, 1);   /* 落定 —— 进入主曲线，如叶落水面 */
  --ease-drift: cubic-bezier(0.45, 0.05, 0.55, 0.95); /* 行云 —— 位置移动、图谱视口 */
  --ease-exit: cubic-bezier(0.4, 0, 1, 1);          /* 收笔 —— 退出，加速离场 */
  --ease-breathe: cubic-bezier(0.37, 0, 0.63, 1);   /* 呼吸 —— 循环动画专用 */
}

@media (prefers-reduced-motion: reduce) {
  :root {
    --motion-instant: 0ms; --motion-fast: 0ms; --motion-base: 0ms;
    --motion-slow: 0ms; --motion-deliberate: 0ms;
  }
  /* 所有位移/缩放动效降级为 120ms 纯 opacity 渐变，循环动画停止 */
}
```

> 与 UI-SPEC v1 §6 的关系：v1 的「进入 100–200ms / 退出 100ms / hover 150ms」被本五档制**收编**——原值均落在 瞬/疾/常 三档附近，迁移即替换为 token 引用。

## 3. 动效编排十律

1. **只动 `transform` 与 `opacity`**（GPU 合成层，保 60fps；图谱 SVG 线条属性除外）。
2. **进慢出快**：进入用 常（220ms）+ 落定，退出用 疾（140ms）+ 收笔。离开的东西不值得目送。
3. **幅度含蓄**：位移 ≤ 16px，缩放 ≥ 0.96。动作如笔锋，点到即止。
4. **雁行错落**：列表/网格进入按序 stagger 25ms，最多 8 项（其后同时出现）——首屏成本封顶 ~400ms。
5. **无弹簧、无回弹、无旋转**（继承 v1「不使用弹簧动效」并扩展）。
6. **循环动画只有「呼吸」**：骨架屏、AI 思考中，opacity 0.5 ↔ 0.85，周期 ≥ 1.8s，呼吸缓动。禁止扫光 shimmer（电商语汇，非文人语汇）。
7. **同刻并发 ≤ 2 组**编排动效；涟漪进行时不叠加其他动画。
8. **空间连续**：侧栏面板从右滑入 16px、从右退出；Dialog 中心缩放 0.96→1。元素从哪来、回哪去。
9. **`prefers-reduced-motion`**：一切位移/缩放/循环停用，仅保留 120ms opacity 渐变。
10. **动效必须携带信息**：说不出「这段动画告诉了用户什么」的动效，删除。

## 4. 图谱画布：水墨视觉语言（核心场景）

### 4.1 静态层（在 graph.css v1 基础上微调）

| 元素 | 设计 | 变化 |
|------|------|------|
| 画布 | 纸。暖白 + 点阵 | 点阵从 neutral-300 **降到 neutral-200 @ 60%**——水痕般淡，退为纸纹 |
| 连线 | 墨线。1.5px bezier 曲线 | 保持；曲线连续性即格式塔连续律，禁用折线 |
| 节点 | 白卡 + 左侧 3px 状态墨条 | 保留 v1 设计；hover 升 shadow-2 |
| 选中 | 2px 靛墨 ring + shadow-2 | 替代加粗描边 |
| 连线标签 | 白底小签 | 保持 |

### 4.2 动效层（新增，四个签名动效）

**① 云雾聚焦（焦点模式）**
选中节点时，图距 > 1 的节点与连线在 320ms（缓/落定）内降至 `opacity 0.22` 并轻微去饱和；相邻一跳保持 0.6；焦点及其直连全亮。退出焦点 140ms 恢复。
*这是「借景」：不给焦点加重，而让远山成雾。*

**② 墨滴涟漪（影响分析）**
影响分析结果按 **BFS 图距逐层点亮**：第 n 跳的节点延迟 `n × 120ms` 开始 220ms 落定动画（描边浸入靛墨 2px ring + 微升 2px——「被影响」是交互强调，用墨色；不用节点自身语义色，避免与状态编码混淆）；连线以 `stroke-dashoffset` 顺传播方向流动一次。整体如墨滴入水。
*峰终定律：传播完成的一瞬是「影响范围」认知落地的峰值时刻；因果知觉：时序差让传播方向不言自明。*

**③ 落印（审批确认）**
节点审批通过时：状态墨条自上而下 220ms wipe 成石绿 → 状态徽章单次脉冲（scale 0.96→1，凝/落定）。如印章落纸，一次即止。
*峰终定律的「终」：给流程一个安静而确定的收束。*

**④ 墨线流动（连线方向）**
hover 或选中某连线时，虚线相位单向流动一次（560ms），指示 TRIGGERS/DEPENDS_ON 的方向语义。静息态永不流动。

### 4.3 视口运动

缩放/定位到节点（如搜索跳转、影响分析入口）：320ms 行云缓动，单段完成，不做多段巡航。

## 5. 组件统一状态层（v2.1 完整矩阵）

所有可交互组件共享一套状态模型，替代各组件散写。八种状态两两之间必须**一眼可辨**，尤其是「可点」与「不可点」——这是 v2.1 修订的起因。

### 5.1 状态矩阵

| 状态 | 视觉表现 | 光标 | 时长/曲线 | 无障碍 |
|------|---------|------|----------|--------|
| rest | 静息，阴影 ≤ shadow-1，变体本色 | default / text | — | — |
| hover | 背景深一档 **或** 阴影升一级（二选一，不叠加）；**必须由 `enabled:` 隔离，禁用态不得触发** | pointer | 疾(140ms) / 落定 | — |
| active | `scale(0.98)`（按钮类）或背景再深一档；`enabled:` 隔离，`motion-reduce` 下取消缩放 | pointer | 瞬(80ms) | — |
| focus-visible | 2px 靛墨 ring + 2px offset；**用 `focus-visible` 而非 `focus`**，避免鼠标点击也描环 | — | 瞬 | 键盘可达焦点 |
| selected | 靛墨边 + 4px 靛墨淡环（`ring-4`）；**边框宽度与 rest 保持一致（均 1px），靠环强调，杜绝选中时 1px 布局跳动** | pointer | 疾 | `aria-selected`/`aria-checked` |
| **disabled「石」** | 见 §5.2 | not-allowed | — | `disabled` 属性 |
| **loading「忙」** | 见 §5.3 | wait | 旋针循环 | `disabled` + `aria-busy` |
| **error「疵」** | 朱砂描边 + 朱砂焦点环，`aria-invalid` 驱动（见 §6.3） | text | 疾 | `aria-invalid` + 关联 caption |

### 5.2 禁用态「石」——形 · 静 · 指 三重信号

> **缺陷复盘**：旧规范用 `opacity 0.45` 表达禁用。品牌紫按钮半透明后仍是「紫底白字」，读起来像可点的按钮只是颜色淡了些；描边/幽灵按钮半透明后几乎无变化。单一「变淡」信号不足以传达「不可点」。

禁用态改为**三重冗余信号**，任一通道失效仍可辨识（前注意加工 + 无障碍）：

1. **形**：所有语义色（品牌紫 / 胭脂 / 靛墨）**整体褪为石灰**——不是降透明度，是换成中性灰底 + 中性灰字（`neutral-100` 底 / `neutral-400` 字，暗色 `neutral-800` / `neutral-600`）。石灰在任何变体上都读作「非活性」，与彩色可点态形成材质级对比。
2. **静**：无任何 hover / active / 动效响应。实现上 hover/active 一律加 `enabled:` 前缀；禁用规则在生成 CSS 中位于 hover 规则之后，确保覆盖。
3. **指**：`cursor: not-allowed`。**注意**：不可再用 `disabled:pointer-events-none`——它会吞掉光标反馈，让用户悬停时得不到「不可」提示。

**各变体「石」化对照**（`@logimap/ui` Button 实现）：

| 变体 | rest（可点） | disabled「石」 |
|------|-------------|---------------|
| default | 品牌紫底白字 | `neutral-100` 底 / `neutral-400` 字 |
| secondary | 浅灰底深字 | 同上（同为中性，靠 cursor + 无 hover 辨识） |
| outline | 白底描边 | `neutral-50` 底 / `neutral-200` 边 / `neutral-400` 字 |
| ghost | 透明底 | 透明底 / `neutral-400` 字，无 hover 浮起 |
| destructive | 胭脂底白字 | `neutral-100` 底 / `neutral-400` 字（危险色必须褪去，否则误读为可点的危险操作） |
| link | 靛墨文字带下划线 | `neutral-400` 字 + 去下划线 |

输入类（Input / Textarea / Select）同理：内凹底切为 `neutral-100`，文字/占位符褪灰，`cursor-not-allowed`；关联 Label 随之褪为 `neutral-400`（不用 opacity）。

**设计准则**：能用「校验反馈」就不要禁用。禁用按钮无法告诉用户「为什么不能点」；优先让控件可点、点击后给出行内校验提示（§6.3）。确需禁用时，附近必须有文案说明原因（如「填写完必填项后可提交」）。

### 5.3 加载态「忙」——忙 ≠ 石

加载是「正在处理」，不是「不可用」，美学上必须与「石」区分：

- **保留变体本色**（品牌紫仍是紫），仅叠加 `Loader2` 旋针 + 轻微降透明（`opacity-90`）。
- 光标 `cursor-wait`（等待），非 `not-allowed`（禁止）。
- 属性上 `disabled + aria-busy=true` 双写：防重复提交，同时向辅助技术播报「忙碌」而非「禁用」。
- 控件保持原尺寸占位，旋针 `shrink-0` 不挤压文字，避免布局跳动。

*依据：Fitts 定律——36px 控件高度保留，图谱节点等画布目标的可点区 ≥ 40×40px；一致的状态语言降低学习成本（Jakob 定律）；状态多通道冗余编码（形+光标+属性）保证色觉障碍与辅助技术用户可达（继承设计库「三重编码」原则）。*

## 6. 三态设计（空 / 载 / 错）

### 6.1 空状态：留白即款待

- 中央窄栏（max-width 320px），上下留白 ≥ 96px。
- 一句引导语，使用 `--font-serif-accent` 衬线——**全产品唯一的衬线落点**，如卷轴题跋：*「纸墨已备，画下第一个节点」*。
- 一个主操作按钮（Hick 定律：空状态只给一条路）。
- 不放插画不放图标堆砌；至多一枚 24px 线性图标。

### 6.2 加载：呼吸骨架

- 骨架块用 `--surface-sunken`（neutral-100），形状忠实于最终内容轮廓（减少布局跳动 = 减少惊吓）。
- 动画为呼吸（律 6），非扫光。
- < 300ms 的请求不出骨架（闪现比等待更糟）；> 1s 补充文字说明。

### 6.3 错误：行内消解

- 表单错误：input/textarea 由 **`aria-invalid=true` 属性驱动**切朱砂描边 + 朱砂焦点环（`aria-[invalid=true]:border-rose-500`），下方配 caption 说明（继承设计库 `is-error`），不弹 toast。属性驱动同时满足语义无障碍——辅助技术播报「无效」，无需额外 class 约定。
- 操作失败 toast：白底 + 胭脂左条 + 具体原因 + 重试动作，8s 自动收笔退出。
- 不满屏红、不感叹号轰炸；错误文案说「发生了什么 + 怎么办」，不道歉不含糊。

## 7. 暗色模式：夜墨

暗色不是反色，是**换纸**——从宣纸换到夜色磁青纸（`#0C0A09` 暖褐黑，已有 token）：

- 层级逻辑不变：背景/表面/浮起三层靠亮度阶表达（已有）。
- 云雾聚焦在暗底改为 `opacity 0.3`（暗底下 0.22 会彻底不可读）。
- 语义色徽章沿用「浅底深字」反转为「深底浅字」：底用 900 档 @ 30% 透明、字用 300 档——需要在实施阶段逐组件验证（设计库 caveat #5 的偿还点）。
- 阴影已切纯黑透明阶（现有 token），保持。
